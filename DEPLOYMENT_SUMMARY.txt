================================================================================
                           QUICK START GUIDE
================================================================================

For Hostinger VPS users, follow these steps:

STEP 1: Read the Quick Start Guide
   Open: HOSTINGER_VPS_QUICKSTART.md
   This will guide you through the entire process

STEP 2: Complete the Pre-Deployment Checklist
   Open: PRE_DEPLOYMENT_CHECKLIST.md
   Fill in all required information before starting

STEP 3: Connect to Your VPS
   ssh root@YOUR_VPS_IP

STEP 4: Install Prerequisites (one-time)
   # Update system
   apt update && apt upgrade -y

   # Install Docker
   curl -fsSL https://get.docker.com -o get-docker.sh
   sh get-docker.sh

   # Install Docker Compose
   curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
   chmod +x /usr/local/bin/docker-compose

   # Install tools
   apt install -y git curl wget nano

STEP 5: Clone Repository
   cd /opt
   git clone YOUR_REPOSITORY_URL SPTracker
   cd SPTracker

STEP 6: Configure Environment
   cp .env.production .env
   nano .env
   # Fill in all CHANGE_ME values with your actual configuration

STEP 7: Deploy
   chmod +x scripts/deploy-vps.sh
   ./scripts/deploy-vps.sh

STEP 8: Access Your Application
   Agent Portal:    http://YOUR_VPS_IP:8080
   Customer Portal: http://YOUR_VPS_IP:8081
   API:             http://YOUR_VPS_IP:5000

================================================================================
                         IMPORTANT CONFIGURATION
================================================================================

Before deploying, you MUST configure these values in .env file:

REQUIRED:
  - POSTGRES_PASSWORD      (generate: openssl rand -base64 32)
  - JWT_KEY                (generate: openssl rand -base64 32)
  - MINIO_ROOT_USER        (your choice)
  - MINIO_ROOT_PASSWORD    (generate: openssl rand -base64 32)
  - SMTP_HOST              (your email provider)
  - SMTP_USER              (your email)
  - SMTP_PASS              (app password)
  - SMTP_FROM              (from address)
  - APP_BASE_URL           (your API URL or http://YOUR_IP:5000)
  - PORTAL_ADMIN_URL       (your agent portal URL or http://YOUR_IP:8080)
  - PORTAL_AGENT_URL       (your customer portal URL or http://YOUR_IP:8081)

OPTIONAL:
  - IMAP_HOST/USER/PASS    (for email-to-ticket feature)
  - TENANT_DEFAULT_TIMEZONE (default: Asia/Dubai)

================================================================================
                          COMMON COMMANDS
================================================================================

View logs:
  docker-compose -f docker-compose.prod.yml logs -f [service]

Restart service:
  docker-compose -f docker-compose.prod.yml restart [service]

Stop all:
  docker-compose -f docker-compose.prod.yml down

Start all:
  docker-compose -f docker-compose.prod.yml up -d

Update app:
  cd /opt/SPTracker
  git pull
  docker-compose -f docker-compose.prod.yml up -d --build

Backup database:
  docker-compose -f docker-compose.prod.yml exec -T postgres \
    pg_dump -U postgres sptrack | gzip > backup.sql.gz

================================================================================
                         TROUBLESHOOTING TIPS
================================================================================

Issue: Port 80 already in use
Fix:   systemctl stop apache2 && systemctl disable apache2

Issue: Container won't start
Fix:   docker-compose -f docker-compose.prod.yml logs [service]

Issue: Can't access from browser
Fix:   ufw allow 80/tcp && ufw allow 443/tcp

Issue: Database connection error
Fix:   Check .env password matches docker-compose.prod.yml

Issue: Email not sending
Fix:   Verify SMTP settings, use app password for Gmail/Office365

Full troubleshooting guide in DEPLOYMENT_GUIDE.md
